## 1. 项目愿景与核心目标

本项目旨在**不模拟生物神经元**，而是提取人类大脑各解剖组织的核心计算原理，构建一个软件系统。

**核心目标：**
- **仿生架构**：系统模块按大脑组织（海马体、杏仁核、额叶、顶下小叶等）划分并命名。
- **并行双通路**：实现快速本能的“皮层下通路”与缓慢深思的“皮层通路”。
- **自主记忆巩固**：具备“清醒-睡眠”循环，通过知识蒸馏将海马体经验转化为皮层长期知识。
- **分层预测驱动学习**：由多认知层级的预测误差驱动学习与注意力分配。
- **自演化世界观**：通过知识图谱和语义规则逐步构建抽象的世界模型。
- **模型灵活且安全配置**：所有LLM后端可通过配置文件或界面动态切换，并经过能力契约校验。

## 2. 系统总体架构

### 2.1 架构设计理念：并行双通路模型
大脑拥有两层处理通路，我们在系统中明确划分：

- **快速通路 (皮层下)**：脑干 → 丘脑 → 杏仁核 / 基底节  
  *职责：本能反应、紧急中断、习惯执行。延迟低、无复杂推理。*
- **慢速通路 (皮层)**：丘脑 → 对应皮层区 (枕叶/颞叶) → 顶叶 (融合与情境预测) → 额叶 (规划与决策)  
  *职责：复杂认知、多步规划、意识级思维。延迟较高但灵活。*

### 2.2 架构图描述
![架构图占位]
```
[外部多模态输入]
        │
        ▼
   [ Thalamus 丘脑 ] ──显著性过滤───
        │                            │
   (快通路)                     (慢通路)
        ▼                            ▼
 [ Amygdala 杏仁核 ]          [ Occipital/Temporal 枕叶/颞叶 ]
        │                            │
        ▼                            ▼
 [ BasalGanglia 基底节]       [ ParietalLobe 顶叶(融合+情境预测) ]
        │                            │
        │                            ▼
        │                      [ FrontalLobe 额叶 ]
        │                            │
        └─────────── [ 行为/语言输出 ] ◄──┘
                      
[ 记忆系统 ]：所有高级情景 → [ Hippocampus 海马体 ] → 离线知识蒸馏 → [ 皮层知识库 ]
```

**数据流关键特征：**
- 所有外部信号都先经过 `Thalamus`。
- `Amygdala` 直接接收高显著信号，可按中断级别暂停皮层处理。
- `ParietalLobe` 基于结构化事件描述发出情境级预测，与真实事件比对，计算语义级 `prediction_error`。
- `prediction_error` 高于阈值的事件被标记为高重要性，存入海马体。
- `Brainstem` 调度“睡眠周期”：海马体记忆重播 → **知识蒸馏** (提取抽象规则) → 更新知识图谱与长期向量库。

### 2.3 生物-模块映射一览表

| 生物组织 | AI 模块名 | 核心职责 | 关键输入 | 关键输出 |
| :--- | :--- | :--- | :--- | :--- |
| 脑干 | `Brainstem` | 系统心跳、分级中断、睡眠/清醒调度 | 时钟信号、任意模块的 `alert` | 全局 `sleep/wake` 信号、`system_alert` |
| 丘脑 | `Thalamus` | 多模态网关、注意力导向、路由分发 | 原始感官数据 | `SensoryPacket` |
| 小脑 | `Cerebellum` | 运动平滑、技能自动化 | 已训练的熟练动作 | 精细控制指令 |
| 杏仁核 | `Amygdala` | 即时情感评估、风险标注、应急响应 | 高显著 `SensoryPacket` | `EvaluatedPacket`，分级中断请求 |
| 海马体 | `Hippocampus` | 情景记忆快速编码、模式分离/完成、记忆重播与抽象提取 | 融合的 `EpisodeTensor` + 重要性评分 | 检索上下文，抽象规则 |
| 基底节 | `BasalGanglia` | 习惯学习与执行、奖励预测 | 熟悉的上下文 | 自动化动作或放行至皮层 |
| 枕叶 | `OccipitalLobe` | 视觉特征提取 (初级→语义) | 视觉 `SensoryPacket` | 结构化视觉描述 |
| 颞叶 | `TemporalLobe` | 听觉/语言处理、语义标注 | 听觉 `SensoryPacket` | 结构化听觉/语义描述 |
| 顶叶 | `ParietalLobe` | 多模态融合、情境预测与误差计算 | 枕叶、颞叶的结构化输出 | `EpisodeTensor`、语义预测误差 |
| 额叶 | `FrontalLobe` | 工作记忆、规划、决策、语言输出 | `EpisodeTensor`、海马体记忆、任务目标进展 | `ActionPlan`、语言响应 |

## 3. 模块详细规格

### 3.0 全局配置模块：`PrefrontalConfig` (动态且安全的模型管理)

所有模块使用的LLM后端均通过该模块动态获取，绝不硬编码，并经过能力校验。

#### 3.0.1 设计目标
- 解耦模型配置与业务代码。
- 运行时热切换模型，无需重启系统。
- 支持 Ollama 本地模型、OpenAI 兼容接口、Anthropic Claude 等多种后端。
- 强制能力契约校验，确保每个脑区调用的模型满足输入/输出格式和推理深度要求。
- 提供简单的配置界面 (使用 Streamlit 或 Gradio)。

#### 3.0.2 配置文件结构 (`config/model_config.yaml`)  【修改】

```yaml
model_registry:
  local-fast:
    provider: ollama
    model_name: "qwen2:0.5b"
    endpoint: "http://localhost:11434"
    scope: ["amygdala"]
    capability_contract:   # 【新增】能力契约
      input_schema:  # 杏仁核期望的输入格式
        type: object
        properties:
          text: {type: string}
      output_schema: # 杏仁核要求的输出格式
        type: object
        properties:
          threat_level: {type: number, minimum: 0, maximum: 1}
          emotion_label: {type: string}
      reasoning_depth: low  # 低推理深度即可

  local-smart:
    provider: ollama
    model_name: "mistral:7b"
    endpoint: "http://localhost:11434"
    scope: ["frontal", "hippocampus-consolidation"]
    capability_contract:
      output_schema: {type: string}  # 额叶规划可输出自由文本或JSON
      reasoning_depth: high

  cloud-deepseek:
    provider: openai_compatible
    model_name: "deepseek-chat"
    endpoint: "https://api.deepseek.com/v1"
    api_key_env: "DEEPSEEK_API_KEY"
    scope: ["frontal", "parietal"]
    capability_contract:
      output_schema: {type: string}
      reasoning_depth: high

  cloud-claude:
    provider: anthropic
    model_name: "claude-3-haiku-20240307"
    api_key_env: "ANTHROPIC_API_KEY"
    scope: ["parietal", "meta-evaluation"]
    capability_contract:
      output_schema: {type: object, properties: {prediction_text: {type: string}, confidence: {type: number}}}
      reasoning_depth: high
```

**规则：**
- `scope` 列表限定该模型可被哪些脑区调用。
- API Key 必须通过环境变量读取，绝不出现在配置文件。
- 每个模型必须定义 `capability_contract`，包含 `output_schema` 等字段。
- 每个脑区模块将声明 `required_capabilities` (如 `reasoning_depth: high`)，**`ModelRouter` 在路由前会进行匹配校验，不满足则拒绝并报错**。

        self.registry = self._load_and_validate(config_path)
        self.active_clients = {}

        # 4. 校验失败则尝试回退或抛出异常

    def switch_model(self, brain_region: str, new_model_id: str):
        """运行时热切换，须再次通过能力校验"""
        ...

    def test_connectivity(self, model_id: str) -> dict:

        ...
    
    def capability_check(self, brain_region_requirements: dict, model_id: str) -> bool:
        """【新增】验证模型能力契约是否满足脑区需求"""
        ...

### 3.1 `Brainstem` (脑干) 【修改：增加中断分级与事务机制】

- **功能**：
  - 维护系统时钟，控制“清醒/睡眠”循环 (如：每60分钟在线，触发10分钟离线巩固)。
  - 接收来自任何模块的 `alert(level, source, message)`，**实施分级中断**：
    - **level < 0.3**：仅记录日志，不中断任何任务。
    - **0.3 ≤ level < 0.7**：向当前活跃的皮层任务发送“减速/自检”信号。
    - **level ≥ 0.7**：立即触发中断，暂停额叶当前任务并**保存工作记忆快照 (回滚状态)**，广播 `EMERGENCY_INTERRUPT`。
  - 中断恢复时，额叶可从快照继续执行。
- **技术实现建议**：
  - 使用 `APScheduler` 或 `asyncio` 定时器，通过事件总线广播 `wake` / `sleep` / `interrupt` 信号。
  - 中断处理：`Brainstem` 维护当前任务的安全阈值 `safety_level`，只有超出阈值才挂起。
- **要求下游模块实现的接口**：
  - `FrontalLobe` 需提供 `rollback_state()` 和 `restore_state(snapshot)` 方法。

### 3.2 `Thalamus` (丘脑)

- **功能**：
  - 接收多模态原始输入 (文本、图像路径、音频片段)。
  - 调用嵌入模型 (`CLIP`, `Whisper`等) 转换为统一向量。
  - 根据当前上下文和杏仁核警报，计算每个输入的 `salience` 分数。
  - 将带有 `salience` 的 `SensoryPacket` 路由到对应的皮层区域，同时根据情况发送拷贝到 `Amygdala`。
- **数据包格式**：
  ```json
  {
    "id": "uuid",
    "timestamp": "ISO8601",
    "modality": "visual/auditory/text",
    "raw_data": ...,
    "embedding": [0.1, 0.23, ...],
    "salience": 0.8,
    "routing": ["OccipitalLobe", "Amygdala"]
  }
  ```
- **技术栈**：
  - 消息队列：`Redis Streams` 或内置 `asyncio.Queue`。
  - 嵌入生成器：预置本地模型，或通过 `ModelRouter` 调用云端嵌入服务。

### 3.3 快速通路

#### `Amygdala` (杏仁核) 【修改：明确输出格式与分级中断】
- **功能**：
  - 接收丘脑转发的高显著包裹，调用专用轻量模型 (由 `ModelRouter` 提供，能力契约要求输出结构化威胁等级)。
  - 输出 `EvaluatedPacket`，**必须包含结构化字段**：`{"threat_level": float, "emotion_label": str}`。
  - 根据威胁等级，向 `Brainstem` 发送相应级别的 `alert`：例如威胁 > 0.7 时发送 level=0.8 的中断。
- **技术**：通过 `ModelRouter.get_client("amygdala")` 获取 LLM，提示词模板要求严格返回JSON。

#### `BasalGanglia` (基底节)
- **功能**：
  - 维护一个“刺激-反应”习惯库：匹配当前上下文 (状态向量) 与已学得的策略。
  - 若匹配置信度高，直接输出封装的动作指令，绕过皮层。
  - 若未匹配，则将状态放行至慢速通路。
- **学习机制**：
  - 在睡眠巩固阶段，从海马体蒸馏出的成功决策模式，可被转换为基底节的习惯规则。
- **接口**：
  ```python
  def habit_lookup(state: Frozenset) -> Optional[Action]:
      ...
  ```

### 3.4 慢速通路 (皮层) 【重大修改：重构预测层级】

#### 3.4.1 `OccipitalLobe` / `TemporalLobe` (枕叶/颞叶) 【修改职责】
- **核心变更**：不再仅输出无结构特征向量，**必须输出结构化的语义描述**。
- **枕叶**：
  - 输入：视觉 `SensoryPacket`。
  - 处理：调用视觉模型 (ViT/CLIP) 识别物体、场景、文字。
  - **输出**：结构化视觉摘要，例如 `{"objects": ["laptop", "coffee cup"], "scene": "office", "text": "..."}`。
- **颞叶**：
  - 输入：听觉 `SensoryPacket`。
  - 处理：语音识别、情感识别、关键词提取。
  - **输出**：结构化听觉摘要，例如 `{"transcript": "...", "keywords": ["meeting", "budget"], "speaker_emotion": "neutral"}`。
- **作用**：为顶叶提供可供逻辑运算的**符号化素材**。

#### 3.4.2 `ParietalLobe` (顶叶) —— 多模态融合与情境预测 【重度修改】
- **核心变更**：彻底重构预测目标，**从“预测下一秒感官数据”改为“预测情境级事件语义”**。
- **功能**：
  1. **多模态融合**：接收枕叶、颞叶的结构化描述，整合为一个统一的事件框架 `{"who": "...", "did_what": "...", "to_whom": "...", "with_tool": "..."}`。可使用规则或小型语言模型填充。
  2. **情境预测生成**：基于当前事件框架和知识图谱 (历史规则)，生成一个对接下来可能发生的事件的预测。例如：“根据历史，当用户询问天气后，有70%概率会追问穿衣建议”。
  3. **预测误差计算**：**使用语义匹配度**。当实际发生的下一事件明确后，计算预测事件与真实事件之间的语义相似度 (例如用Sentence-BERT计算两个描述句子的相似度)。**`prediction_error = 1 - semantic_similarity`**。
  4. **重要性标记**：当 `prediction_error > 阈值 (如 0.6)` 时，表明出现了“意外”，将当前 `EpisodeTensor` 的重要性评分设为高。
- **技术实现**：
  - 事件框架生成：利用 LLM 按指定JSON模式填充。
  - 预测模型：可以是一个简单的统计模型 (如Markov链) 或基于知识图谱的推理器，而非端到端深度生成模型，避免计算爆炸。
  - 误差计算：调用轻量级句子嵌入模型。
- **模块规格补充**：
  > *“顶叶接收枕叶/颞叶输出的结构化事件描述，生成情境级预测。预测误差 = 1 - 语义匹配度。仅当误差 > 0.6 时，额外提升该记忆的重要性并存入海马体。”*

#### 3.4.3 `FrontalLobe` (额叶) —— 中央执行系统 【修改：引入抽象层误差与回滚】
- **功能**：
  - **工作记忆**：维护当前任务状态、对话历史、临时子目标，并能够通过 `rollback_state()` 保存快照，由 `Brainstem` 中断触发。
  - **规划**：接收 `ParietalLobe` 的融合事件及海马体检索的相关记忆，调用 LLM 分解任务和规划步骤。
  - **抽象层误差监控**：计算“当前状态 vs 规划路径的偏离度”。例如，规划要求从用户获取三个信息，但两轮后用户情绪转为愤怒，偏离规划，触发内部修正或升级。
  - **决策输出**：产生 `ActionPlan`。
- **技术实现**：
  - 通过 `ModelRouter.get_client("frontal")` 获取 LLM。
  - 工作记忆使用 `deque` + 任务树结构，需实现序列化/反序列化以支持快照。

### 3.5 记忆系统：从记忆巩固到知识蒸馏 【重度修改】

#### `Hippocampus` (海马体)
- **功能**：
  - **快速编码**：存储来自 `ParietalLobe` 的 `EpisodeTensor` (包含结构化事件、情感标签、重要性评分、预测误差等)。
  - **重要性计算**：结合 `prediction_error`、杏仁核威胁等级及任务相关性，自动计算 `importance_score`。
  - **检索**：根据当前情景嵌入，查询最相似的K个记忆。
- **技术**：ChromaDB 或 Milvus。

#### 记忆巩固 (模拟睡眠) —— **严格限定为知识蒸馏** 【核心修改】
- **触发**：由 `Brainstem` 发送 `sleep_cycle` 信号。
- **流程**：**严禁任何形式的LLM参数微调 (Fine-tuning/LoRA)**。巩固被重新定义为：
  1. **采样**：从海马体取出 `importance_score > 0.8` 的事件。
  2. **抽象提取** (`extract_abstraction`)：使用轻量级NLP管道 (如正则+语义角色标注，或一个专门的LLM调用) 从每个事件中提取**抽象规则或常识**。  
     *示例：* 从“用户说‘紧急’+‘退款失败’+愤怒情绪”一系列事件中，提取出规则： `IF 情绪=愤怒 AND 关键词包含退款失败 THEN 建议触发安抚与人工转接协议`。
  3. **更新外部知识存储** (非权重)：
     - 将提取出的**规则**写入知识图谱 `knowledge_graph.json` (作为节点和关系)。
     - 将规则的文本描述及嵌入存入长期语义向量库 (用于未来的RAG检索)。
  4. **强化习惯**：某些规则可转化为基底节的直接映射。
- **算法实现要求**：
  - `memory/consolidation.py` 中不得出现 `model.train()`、`LoRA`、`PEFT` 等与神经网络参数更新相关的代码 (除非未来集成了完全独立的小型皮层预测网络)。
  - `extract_abstraction()` 方法必须在文档中有伪代码定义：
    > *“接收一个事件 JSON，使用 heuristics (如基于模板的规则提取) 或调用一个已配置的 LLM，提示词为‘从以下客户服务对话中提取一条 if-then 业务规则...’，输出纯文本规则。”*

## 4. 全局数据流与状态机 【小幅修改】

### 4.1 清醒期 (Online Phase)
```
外部输入 → Thalamus
   │
   └→ (快) → Amygdala → 若威胁等级高 → Brainstem分级中断 → 应急/暂停
   │                   ↓ (无中断或低级别)
   └→ (慢) → 枕叶/颞叶 结构化感知
                │
                ▼
            ParietalLobe 融合为事件框架 → 生成情境预测
                │
                ├→ 计算语义预测误差
                ├→ 输出 EpisodeTensor (带误差)
                │
                ▼
            FrontalLobe 规划
                │
                ├→ 查询海马体
                ├→ 监控任务偏离
                └→ 输出动作
```
- **存储**：`EpisodeTensor` (含误差和重要性) 流入 `Hippocampus`。

### 4.2 睡眠期 (Offline Phase)
```
Brainstem sleep 信号
   │
   └→ Hippocampus 重播高重要性事件
          │
          ├→ 提取抽象规则 (知识蒸馏)
          ├→ 更新知识图谱 / 长期语义向量库
          └→ 转化习惯入基底节
   │
   └→ 修剪低重要性陈旧记忆
   │
   └→ Brainstem 发出 wake 信号
```

## 附录：工程实施注意事项 (基于Gemini专业评估反馈)

在开发过程中，需特别关注以下潜在瓶颈，但无需修改上述架构规格：

1. **皮层慢通路的“延迟级联”**：慢通路 (丘脑→枕/颞叶→顶叶→额叶) 若串行调用多个LLM，延迟会累加。**对策**：确保枕叶/颞叶处理可并行；基底节习惯匹配作为快速旁路；MVP阶段可先合并颞叶与顶叶的部分调用以减少跳数。
2. **额叶状态快照的工程复杂度**：中断时若额叶正等待LLM流式响应，难以完美回滚。**对策**：初版实现中，回滚仅针对已完成的工作记忆步骤，正在生成的流直接丢弃并基于上一个完整快照重新规划。
3. **语义预测误差的固定阈值**：阈值0.6可能不通用。**对策**：在`ParietalLobe`中预留`adaptive_threshold`参数，初期硬编码，后期可接入简单的移动平均或强化学习动态调整。
